# Unverified Claims

This section prevents unsafe ZK claims from leaking into the canonical theory.

## Must Not Claim Yet

- complete production ZK security
- audited circuit safety
- full Ethereum production readiness
- global address predicate coverage
- universal verifier interoperability
- private delivery safety without an operational threat model

## Safe Wording

- ZK-ready
- predicate-compatible
- proof envelope model
- public signal policy
- circuit profile pending audit
- external audit required before production use

## Required Evidence Before Stronger Claims

1. circuit implementation
2. circuit test vectors
3. public signal leak tests
4. witness handling policy
5. verifier policy review
6. cryptographic review
7. operational threat model
8. revocation and freshness root governance
