# 2. Address Basics: Reference, Compression, Communication, History

An address has at least four roles.

- **Reference:** it points to an addressable entity.
- **Compression:** it compresses local knowledge into a portable expression.
- **Communication:** it lets humans, machines, and institutions coordinate.
- **History:** it carries lineage across renaming, subdivision, merger, and
  deprecation.

AMT treats these roles separately. A shorter label can be good compression but
poor evidence. A coordinate can be good geometry but poor social reference.
A postal code can be good routing metadata but insufficient for registration.

Model hook: `formal/entropy.ts`, `formal/history-graph.ts`.
