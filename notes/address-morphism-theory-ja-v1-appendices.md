# 付録A 中核記法

本付録では、本文で使う主要記法をまとめる。日本語版v1では直感的説明を優先するが、英語国際論文版へ変換するときは、この記法表を定義部の基礎にする。

| 記法 | 意味 | 備考 |
| --- | --- | --- |
| \(t\) | 時点 | 住所体系、地物、行政区画、利用文脈は時間に依存する |
| \(W_t\) | 時点 \(t\) の世界状態 | 物理地理、行政制度、社会的状態、データ出典を含む |
| \(E_t\) | 住所可能実体集合 | 建物、土地、部屋、道路、橋、川、湖、島、遺跡、仮想住所など |
| \(S_t\) | 表面住所表現集合 | 人間が入力・表示する文字列、コード、座標、混合表現 |
| \(C\) | 利用文脈 | 配送、行政、消防、不動産、観光、災害支援など |
| \(O_t\) | 観測写像 | 実体から観測可能な表面表現や特徴への写像 |
| \(\pi\) | 解析写像 | 入力表現をトークン、言語、階層、地物型へ分解する |
| \(\epsilon_t\) | 展開写像 | 別名、旧称、翻訳、略称、近傍候補を展開する |
| \(\Gamma_t\) | 候補生成写像 | 入力から候補実体集合を生成する |
| \(D_t\) | 構造的非類似度 | 地理、行政、属性、型、履歴、文脈の差を測る |
| \(\Pi_{\delta,t}\) | クラスタ分割 | 閾値 \(\delta\) に基づく候補集合の分割 |
| \(Q_t\) | 品質評価関数 | 出典、鮮度、履歴、候補被覆、粒度、リスクなどの評価 |
| \(R_t\) | 解決結果型 | resolved, ambiguous, unresolved, rejected, conditional |
| \(H_t\) | 履歴グラフ | 住所、実体、PID、制度変更の時間的関係 |
| PID | 永続参照識別子 | AMT本体の解決ゲートを通過した参照に発行する |

# 付録C 反例集

| 反例 | 内容 | 影響 |
| --- | --- | --- |
| 同名ビル | 同一都市や近隣地域に同名の建物が複数存在する | 表面名だけでは解決できない |
| 同一郵便番号内の多数住宅 | 郵便番号が区域を表し、単一建物や住戸を表さない | 郵便番号だけでPIDを発行してはならない |
| 垂直衝突 | 同一座標に1階店舗、2階事務所、地下施設が重なる | 2次元座標だけでは不十分 |
| 旧住所衝突 | 行政変更後も旧住所が利用される | 履歴グラフが必要 |
| 自然地名衝突 | 川、山、湖、道路、橋、集落名が同一または類似名を持つ | 地物型と範囲が必要 |
| 社会的移転 | 会社や店舗が移転しても同一主体として扱われる場合がある | 地理的同一性だけでは不十分 |
| 災害時住所 | 避難所、仮設住宅、支援拠点が短期間で変化する | 一時住所と鮮度管理が必要 |
| 世界遺産・遺跡 | 文化的範囲、管理範囲、観光入口が一致しない | 参照粒度を分ける必要がある |
| 砂漠・湿地・氷原 | 境界が曖昧または季節で変化する | 面状地物の品質と季節性が必要 |

# 付録F 英語国際論文版への変換規則

日本語版v1は母艦原稿であり、英語版は逐語訳ではなく国際論文として再構成する。英語版では次の方針を採る。

1. 主張を強くしすぎない。検証済みの数学的主張、検証計画、実装可能なモデルを中心にする。
2. AMT本体とAGID/AOID応用論文を分離する。
3. AMT本体とZKP住所述語論文を分離する。
4. 反例を先に置き、なぜ通常の住所正規化では足りないかを明確にする。
5. 定義、補題、命題、定理、系を番号付きで整理する。
6. 可換図式は本文の理解を助ける場所に置く。
7. 商用APIとの比較では「常に勝つ」と書かず、評価軸と対象データを明示する。
8. 自然地理や文化地理は応用範囲として扱い、完全性を主張しない。
9. 品質スコアは内部判定であり、利用者へ数値を直接表示しない方針を明記する。
10. 検証できない期待は、期待、仮説、検証計画、失敗時の書き換えに分ける。

英語版の仮題は次のいずれかがよい。

| 候補 | 意図 |
| --- | --- |
| Address Morphism Theory | 最も簡潔な基本題名 |
| Address Morphism Theory: A Formal Framework for Ambiguous, Historical, and Contextual Address Reference | 国際論文向けに内容を説明する題名 |
| Address Morphism Theory: Safe Address Resolution under Ambiguity, History, and Context | 安全な解決と不可能性定理を前面に出す題名 |

英語版では、本文の中心を「完全解決を主張する理論」ではなく、「完全解決不能性を認めた上で安全な解決と保留を設計する理論」として提示する。

# 付録G 可換図式挿入計画

可換図式は、本文の理解を助ける場所に限定して置く。日本語版v1では図式案として保持し、PDF化または英語国際論文版では LaTeX の `tikz-cd` 形式へ変換する。

## G.1 AMT写像連鎖

挿入位置: 第7章 7.1 の後。

目的: 住所解決が一回の変換ではなく、解析、展開、候補生成、クラスタ、評価、解決結果、PID発行、履歴更新の連鎖であることを示す。

```text
S_t
  -- pi -->
Token_t
  -- epsilon_t -->
Expanded_t
  -- Gamma_t -->
Candidates_t
  -- Pi_delta,t -->
Clusters_t
  -- Eval_c,t -->
R_t
  -- Gate -->
PID or NonIssue
  -- Update -->
H_{t+1}
```

## G.2 非単射観測と不可能性

挿入位置: 第6章 6.2 の後。

目的: 異なる実体が同じ観測へ潰れると、観測から実体への一意な逆写像が存在しないことを示す。

```text
x1 in E_t  -- O_t --> y
x2 in E_t  -- O_t --> y
x1 != x2

No resolver F: Y_t -> E_t can satisfy
F(O_t(x)) = x
for all x in E_t.
```

## G.3 表面表現、実体、PID、履歴

挿入位置: 第11章 11.8 の後。

目的: PIDが住所文字列ではなく、解決手続きと履歴状態を通過した参照であることを示す。

```text
Surface expression
  --> Candidate cluster
  --> Resolution gate
  --> PID
  --> History graph

Surface expression
  --> History graph
  --> Candidate cluster
```

## G.4 文脈相対性

挿入位置: 第14章 14.3 の後。

目的: 同じ入力でも、配送、行政、消防、不動産で最適出力が変わることを示す。

```text
          context c1 --> result r1
input s <
          context c2 --> result r2

In general, r1 != r2.
```

## G.5 履歴グラフと住所保存則

挿入位置: 第12章 12.4 の後。

目的: 住所が単に消えるのではなく、旧住所、新住所、分割、統合、訂正、PID継承へ写像される場合があることを示す。

```text
Old address node
  -- rename / administrative change -->
New address node

Old PID
  -- inherit / split / merge / retire -->
New PID state
```

## G.6 AMTと暗号拡張の分離

挿入位置: 第18章 18.6 の置換候補。

目的: AMTが意味論を提供し、ZKPやcredentialが秘匿証明を提供するという境界を示す。

```text
Address input
  --> AMT semantic pipeline
  --> resolved class / attributes / audit envelope
  --> cryptographic predicate
  --> ZK proof
  --> public statement
```

## G.7 自然地理とアクセス点

挿入位置: 第16章 16.8 の後。

目的: 山、湖、川、島、遺跡などでは、地物そのもの、表示点、アクセス点、配送可能点を分ける必要があることを示す。

```text
Named geographic feature
  --> geometry / extent
  --> label point
  --> access point
  --> context-specific addressable reference
```
