# 住所写像論 日本語版 v1 章立て案

## 方針

本稿は「住所写像論」の日本語版を母艦として再構成するための章立て案である。英語版はこの日本語版を単純翻訳するのではなく、国際論文として再編集する。したがって日本語版では、原文の直感、発想の順序、例、反例、言葉の強さを優先し、その上で必要な数理モデルと検証可能な定理を差し込む。

現行の英語草稿やZKP草稿は、理論仕様書としては有用だが、原文の思想から離れすぎる危険がある。本v1では、AMT本体、ZKP、AGID/AOID応用を明確に分離する。

## 原文準拠で残す中核

1. 住所は単なる文字列ではない。
2. 住所は単なる座標でもない。
3. 住所は、人間社会が空間・制度・履歴・配送可能性を圧縮して扱うための参照である。
4. 住所解決は、文字列から一点を返す処理ではなく、候補生成、展開、クラスタリング、評価、保留、履歴更新、識別子発行の連鎖である。
5. 非単射観測と候補不完全性の下では、条件なしの完全住所解決器は存在しない。
6. したがって `unresolved`、`ambiguous`、`rejected` は失敗ではなく安全状態である。
7. 住所は現在値だけでなく履歴グラフとして扱うべきである。
8. 住所の最適表現は配送、行政、消防、不動産、災害支援などの文脈に依存する。
9. 郵便番号、従来住所、座標コード、Plus Codes、PIDなどは住所情報の異なる圧縮方式として比較できる。
10. AGID/AOIDやZKPは重要な応用だが、AMT本体の定義そのものではない。

## 推奨タイトル

住所写像論

副題:

曖昧な住所解決、履歴、保留状態、永続参照のための数理モデル

英語版候補:

Address Morphism Theory: A Mathematical Model of Ambiguous Address Resolution, Lineage, Abstention, and Persistent Reference

## 全体構成

### 第1章 序論 - 住所とは何が難しいのか

目的:

原文の問題意識をそのまま立ち上げる章。いきなり定義しない。まず、住所がなぜ単なる文字列でも座標でもないかを具体例で示す。

入れる内容:

- 同じ場所に複数の住所表記がある。
- 同じ住所表記が複数候補を指す。
- 建物、部屋、入口、ロッカー、階層が2次元座標では衝突する。
- 道路、橋、山、川、湖、島、遺跡、世界遺産なども名前付き参照になりうる。
- 災害、行政変更、再開発、移転で住所が変わる。
- 解決できないときに無理に一点を返すことが危険である。

主張:

住所解決は「文字列から座標」ではなく、「曖昧な表現から、文脈に応じて安全に参照を形成する問題」である。

### 第2章 住所の基本仮説 - 参照、圧縮、通信、履歴

目的:

住所写像論の直感的な核を置く章。数式より先に概念を固定する。

入れる仮説:

- 住所は参照である。
- 住所は情報圧縮である。
- 住所は通信先である。
- 住所は現在値ではなく履歴を持つ。
- 住所は社会的実体とも結びつく。

注意:

「DIDより根源的」などの強い仮説は本章では予想として置き、定理のように断定しない。

### 第3章 住所可能実体

目的:

AMTが扱う対象を広げる章。ここで「住所=建物住所」から解放する。

対象:

- 住宅、店舗、倉庫、学校、役所
- 建物、入口、階、部屋、ロッカー
- 道路、橋、駅、港、空港
- 川、滝、池、湖、島、山、谷、洞窟
- 砂漠、草原、森林、湿地、塩湖、氷河、氷原
- 遺跡、世界遺産、文化財
- 避難所、仮設住宅、災害支援拠点
- 私書箱、宅配ロッカー、仮想住所

主張:

住所可能性は郵便配送可能性より広い。配送不能な対象でも、参照可能な地理・社会・文化実体であればAMTの対象になりうる。

### 第4章 表面住所表現

目的:

入力される住所表現を定義する章。

入れる内容:

- 通常住所
- 省略住所
- 多言語住所
- 旧住所
- 建物名
- 自然地名
- 施設名
- 郵便番号
- 座標
- Plus Codes
- PID
- AGID/AOID参照
- 混合表現

主張:

表面表現は実体そのものではない。表面表現は観測であり、観測は非単射でありうる。

### 第5章 形式的準備

目的:

第1章から第4章までの直感を、最小限の数理モデルに落とす。

定義:

- 表面住所表現集合 `S_t`
- 住所可能実体集合 `X_t`
- 観測写像 `O_t`
- 候補生成写像 `Gamma_{c,t}`
- 構造的非類似度 `D_{c,t}`
- クラスタ分割 `Pi_{delta,c,t}`
- 評価関数 `E_{c,t}`
- 解決結果 `Resolved / Ambiguous / Unresolved / Rejected`
- 永続識別子 `PID`
- 履歴グラフ `G_H`

執筆方針:

数式を入れるが、先に例を置く。読者が「何のための定義か」を失わないようにする。

### 第6章 住所参照不可能性定理

目的:

本論文の主定理を置く章。

中心定理:

非単射観測、候補欠落、射影損失、または文脈衝突が存在する場合、条件なしの完全住所解決器は存在しない。

入れる補題:

- 非単射観測補題
- 候補欠落補題
- 正規化衝突補題
- 2次元射影衝突補題
- 郵便番号衝突補題

入れる反例:

- 同名ビル
- 同一郵便番号内の多数住宅
- 同一座標上の1階店舗と2階事務所
- 行政変更後の旧住所
- 自然地名と行政地名の衝突

結論:

完全解決不能性は弱点ではなく、AMTに保留状態が必要であることの根拠である。

### 第7章 AMT写像連鎖

目的:

住所解決を一回の変換ではなく、連鎖として定義する章。

基本連鎖:

`表面表現 -> 解析 -> 展開 -> 候補生成 -> 構造比較 -> クラスタ -> 評価 -> 解決/保留 -> PID発行/非発行 -> 履歴更新`

各節:

- 解析
- 展開
- 候補生成
- 構造的非類似度
- クラスタリング
- 評価
- 解決結果

注意:

ここではZKPやAGID/AOID詳細に行かない。AMT中核のパイプラインだけを書く。

### 第8章 候補生成と出典政策

目的:

候補生成を独立章に格上げする。原文の「真の対象を早く捨てない」思想を中心に置く。

入れる内容:

- 公式郵便ソース
- 行政データ
- オープン地理データ
- 地図データ
- 建物データ
- 自然地理データ
- 歴史・旧住所データ
- 配送履歴
- ユーザー補助情報
- 多言語・別名展開

主張:

候補生成では精度より再現率を優先する。誤候補は後段で落とせるが、真候補を候補集合に入れなければ後段は回復できない。

### 第9章 クラスタと住所同値類

目的:

同じ実体を指す表記揺れ、別名、旧名、多言語表記、階層差を扱う章。

入れる内容:

- 住所同値類
- クラスタの文脈依存性
- 建物レベル同値と部屋レベル非同値
- 自然地名の同値性
- 旧住所と新住所の対応

主張:

表記よりも同値類の方が安定する場合がある。ただし同値類は絶対ではなく、文脈・時刻・証拠に依存する。

### 第10章 未解決状態の理論

目的:

`unresolved` をAMTの中心概念として扱う章。

入れる状態:

- `resolved`
- `ambiguous`
- `unresolved`
- `rejected`
- `conditional`

主張:

未解決は失敗ではなく、安全な出力である。証拠が不足しているときにPIDを発行しないことが、住所写像論の重要な設計原則である。

### 第11章 安全な解決とPID発行

目的:

「候補を表示すること」と「PIDを発行すること」を分ける章。

入れる内容:

- PID発行許容述語
- 候補被覆証拠
- 一意クラスタ
- 品質しきい値
- 鮮度
- リスク予算
- 監査ハッシュ
- PID発行証明書

主張:

PIDは最善推測ではない。AMTゲートを通過した監査可能な出力である。

### 第12章 履歴グラフと住所保存則

目的:

住所を現在値ではなく履歴グラフとして扱う章。

入れる内容:

- 行政変更
- 町名変更
- 再開発
- 建物建替え
- 分割
- 統合
- 移転
- 災害後の仮想住所
- PID継承

主張:

住所は単純に消滅するのではなく、写像、分岐、統合、失効、継承として記録される。これを「住所保存則」として、強い定理ではなく原理または運用公理として置く。

### 第13章 社会的連続性

目的:

住所と社会的実体の関係を扱う章。

例:

- 会社が移転しても同じ会社である。
- 店舗が移転してもブランドや営業主体が続く。
- 避難者の通常住所が失われても支援対象としての同一性は続く。

主張:

住所の同一性は地理だけで決まらない場合がある。社会的連続性、制度的連続性、所有・利用・配送履歴が関与する。

注意:

この章は挑戦的なので、定理ではなく仮説・モデルとして書く。

### 第14章 文脈相対的最適性

目的:

住所相対性原理を置く章。

文脈:

- 配送
- 消防
- 警察
- 行政
- 不動産
- 税務
- 災害支援
- ロボット配送
- 一般検索

中心命題:

すべての文脈で同時に最適な住所表現は一般には存在しない。

主張:

住所の正しさは単一ではなく、目的関数に依存する。

### 第15章 住所圧縮と住所エントロピー

目的:

郵便番号、住所、座標、Plus Codes、PIDを比較する章。

入れる内容:

- 住所は情報圧縮である。
- 郵便番号は配送経路圧縮である。
- 座標コードは幾何圧縮である。
- 従来住所は社会的・言語的圧縮である。
- PIDはAMTゲート通過後の参照圧縮である。
- エントロピーは候補集合の不確実性を測る。

注意:

「都市化とともに住所情報量が増える」は仮説として扱う。一般定理にしない。

### 第18章 AMTと暗号拡張の境界

目的:

ZKPを別論文に分けるための境界章。

書くこと:

- AMTは住所意味論である。
- ZKPはAMT由来属性を秘匿証明する応用層である。
- AMTだけでは暗号安全性を保証しない。
- ZKPだけでは住所の真実性を保証しない。

本文に残す範囲:

ZKPの詳細ではなく、ZKPが消費できるAMT出力を説明する。

外すもの:

- nullifier詳細
- revocation root
- proof bundle
- AOID ownership proof
- delivery eligibility proof
- ZK circuit安全性

### 第19章 PIDと応用識別子の境界

目的:

AGID/AOIDを別論文に分けるための境界章。

- 公開地理参照、私的配送情報、QR、API、SDK、通信境界は応用論文に送る。

### 第25章 限界

目的:

書くこと:

- AMTは暗号安全性を保証しない。
- 品質スコアは真理ではない。

- 自然地理名は境界が曖昧な場合がある。

- AGID/AOID/ZKPは別の実装監査が必要である。

### 第26章 結論

目的:

原文の核に戻って締める。

結論:

住所写像論は、住所を単なる文字列や座標ではなく、圧縮され、履歴を持ち、文脈に依存し、社会制度と結びつく参照として扱う理論である。非単射観測と候補不完全性により、条件なしの完全住所解決器は存在しない。したがって安全な住所システムは、解決だけでなく、曖昧、未解決、拒否、履歴更新、監査可能PID発行を扱わなければならない。
